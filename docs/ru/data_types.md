## Базовые типы данных

### Integer (Целое число)

- **Внутреннее представление**: `f64` (число с плавающей точкой, но дробная часть равна 0)
- **Описание**: Целые числа (42, -10, 0)
- **Примеры**:
  ```datacode
  global x = 42
  global y = -100
  global z = 0
  ```

**📚 Примеры:** [`examples/ru/01-основы/arithmetic.dc`](../../examples/ru/01-основы/arithmetic.dc)

### Float (Число с плавающей точкой)

- **Внутреннее представление**: `f64`
- **Описание**: Числа с десятичной частью
- **Примеры**:
  ```datacode
  global pi = 3.14
  global price = 99.99
  global temperature = -5.5
  ```

**Примечание**: Integer и Float совместимы друг с другом в операциях и считаются числовыми типами.

**📚 Примеры:** [`examples/ru/01-основы/arithmetic.dc`](../../examples/ru/01-основы/arithmetic.dc)

### String (Строка)

- **Внутреннее представление**: `String`
- **Описание**: Текстовые данные
- **Примеры**:
  ```datacode
  global name = 'Hello'
  global message = 'DataCode is awesome'
  global empty = ''
  ```

Строки поддерживают **интерполяцию**: подстановку выражений в формате `${expr}`, вывод «имя=значение» (`${x=}`) и форматирование чисел (`${n:.2f}`). Подробнее: [Форматирование строк (интерполяция)](./string_format.md).

**📚 Примеры:** [`examples/ru/01-основы/strings.dc`](../../examples/ru/01-основы/strings.dc)

### Bool (Логический тип)

- **Внутреннее представление**: `bool`
- **Описание**: Логические значения `true` или `false`
- **Примеры**:
  ```datacode
  global is_active = true
  global is_deleted = false
  ```

**📚 Примеры:** [`examples/ru/02-синтаксис/booleans.dc`](../../examples/ru/02-синтаксис/booleans.dc)

### Null

- **Внутреннее представление**: специальное значение `Value::Null`
- **Описание**: Отсутствие значения или неопределенное значение
- **Примеры**:
  ```datacode
  global value = null
  ```

**Особенности**:
- `Null` совместим с любым типом данных
- Используется по умолчанию для неинициализированных переменных

## Составные типы данных

### Array (Массив)

- **Внутреннее представление**: `Vec<Value>`
- **Описание**: Упорядоченная коллекция значений любого типа
- **Примеры**:
  ```datacode
  global numbers = [1, 2, 3, 4, 5]
  global mixed = [1, 'hello', true, null]
  global nested = [[1, 2], [3, 4]]
  ```

**Особенности**:
- Массивы могут содержать элементы разных типов
- Поддерживается вложенность (массивы массивов)
- Доступ к элементам по индексу: `arr[0]`

**📚 Примеры:** [`examples/ru/01-основы/`](../../examples/ru/01-основы/)

### Object (Объект/Словарь)

- **Внутреннее представление**: `HashMap<String, Value>`
- **Описание**: Неупорядоченная коллекция пар ключ-значение
- **Примеры**:
  ```datacode
  global person = {
    'name': 'Alice',
    'age': 30,
    'active': true
  }
  ```

**Особенности**:
- Ключи всегда строки
- Значения могут быть любого типа
- Доступ к значениям через точку: `person.name` или `person['name']`

## Специальные типы для работы с таблицами

### Table (Таблица)

- **Внутреннее представление**: `Rc<RefCell<Table>>`
- **Описание**: Структурированные данные с колонками и строками
- **Примеры**:
  ```datacode
  global data = read_file('data.csv')  # read_file может загружать csv, excel таблица, возвращает table
  ```
  ```datacode
  global table1 = table(data, ['name', 'age', 'salary'])
  ```

**Особенности**:
- Автоматическое определение типов колонок
- Оптимизированное хранение для больших объемов данных
- Поддержка операций фильтрации, сортировки, выборки

**Методы таблицы**:
- `table.rows` - доступ к строкам
- `table.columns` - доступ к списку колонок
- `table.idx[i]` - доступ к строке по индексу
- `table['column_name']` - доступ к колонке по имени

**📚 Примеры:** 
- [`examples/ru/09-создание модели данных/`](../../examples/ru/09-создание%20модели%20данных/)
- См. также: [Работа с таблицами](./table_create_function.md)

### TableColumn (Колонка таблицы)

- **Внутреннее представление**: `(Rc<RefCell<Table>>, String)`
- **Описание**: Ссылка на колонку таблицы с указанием имени
- **Примеры**:
  ```datacode
  global column = table.name  # Получить колонку 'name'
  ```

**Особенности**:
- Представляет коллекцию значений определенной колонки
- Сохраняет информацию о типе данных колонки

### TableIndexer (Индексатор таблицы)

- **Внутреннее представление**: `Rc<RefCell<Table>>`
- **Описание**: Специальный объект для доступа к строкам таблицы по индексу
- **Примеры**:
  ```datacode
  global first_row = table.idx[0]
  ```

**Особенности**:
- Используется для доступа к строкам через `.idx[i]`
- Возвращает массив значений строки

### LazyTable (Ленивая таблица)

- **Внутреннее представление**: `LazyTable`
- **Описание**: Таблица с отложенным выполнением операций
- **Примеры**:
  ```datacode
  global filtered = table.filter(x > 10)
  ```

**Особенности**:
- Операции не выполняются сразу, а накапливаются
- Выполнение происходит только при материализации (materialize)
- Поддерживаемые операции: `filter`, `where`, `select`, `head`, `tail`, `sort`

## Специальные типы для работы с файлами

### Path (Путь к файлу)

- **Внутреннее представление**: `PathBuf`
- **Описание**: Путь к файлу или директории
- **Примеры**:
  ```datacode
  global file_path = path('/path/to/file.csv')
  global files = list_files('/path/to/dir')  # Возвращает массив Path объектов
  ```
  
**Особенности**:
- Функция `list_files()` возвращает массив объектов `Path` с полными путями к файлам и директориям
- Path объекты можно использовать напрямую с функциями `read_file()`, `path()` и другими
- Функция рекурсивно обходит все поддиректории по умолчанию
- Опциональный параметр `regex` для фильтрации файлов по шаблону имени (поддерживает glob-паттерны типа `*.csv` или обычные регулярные выражения)

**Методы Path объекта**:

| Метод | Тип возвращаемого значения | Описание |
|-------|---------------------------|----------|
| `name` | String | Имя файла/директории (последний компонент пути) |
| `parent` | Path или Null | Родительская директория |
| `exists` | Bool | Проверка существования файла/директории |
| `is_file` | Bool | Является ли путь файлом |
| `is_dir` | Bool | Является ли путь директорией |
| `extension` | String | Расширение файла (без точки), пустая строка если нет расширения |
| `len` | Number | Длина строкового представления пути |

**Примеры использования методов**:
```datacode
global file = path('/home/user/document.txt')

print(file.name)        # 'document.txt'
print(file.parent)      # Path('/home/user')
print(file.exists)      # true/false
print(file.is_file)     # true
print(file.is_dir)      # false
print(file.extension)   # 'txt'
print(file.len)         # 25

# Использование с list_files()
global files = list_files('/path/to/dir')
for file in files {
    if file.is_file and file.extension == 'csv' {
        print('CSV file:', file.name)
        global data = read_file(file)  # Прямое использование Path объекта
    }
}

# Использование с фильтром regex (glob-паттерны)
global csv_files = list_files('/path/to/dir', regex='*.csv')
for file in csv_files {
    print('CSV файл:', file.name)
}

# Использование с фильтром regex (несколько расширений)
global data_files = list_files('/path/to/dir', regex='*.csv|*.xlsx')
for file in data_files {
    print('Файл данных:', file.name)
}

# Использование с фильтром regex (обычное регулярное выражение)
global numbered_files = list_files('/path/to/dir', regex='^file\\d+\\.txt$')
for file in numbered_files {
    print('Пронумерованный файл:', file.name)
}
```

**📚 Примеры:** 
- [`examples/ru/01-основы/`](../../examples/ru/01-основы/)
- [`examples/ru/09-создание модели данных/01-file-operations.dc`](../../examples/ru/09-создание%20модели%20данных/01-file-operations.dc)

### PathPattern (Шаблон пути)

- **Внутреннее представление**: `PathBuf`
- **Описание**: Glob-паттерн для поиска файлов
- **Примечание**: Также можно использовать параметр `regex` в `list_files()` напрямую для фильтрации файлов
- **Примеры**:
  ```datacode
  global pattern = path('/path/*.csv')
  global files = list_files(pattern)
  
  # Или использовать параметр regex напрямую
  global files = list_files('/path/to/dir', regex='*.csv')
  ```

## Распознавание и преобразование типов

### Date (Дата)

- **Обнаружение**: Автоматически определяется из строк при загрузке данных
- **Поддерживаемые форматы**:
  - ISO: `YYYY-MM-DD` (например, `2023-12-25`)
  - RFC3339: `YYYY-MM-DDTHH:MM:SSZ` (например, `2023-12-25T10:30:00Z`)
  - Европейский: `DD.MM.YYYY` или `DD/MM/YYYY` (например, `25.12.2023`, `25/12/2023`)
  - Американский: `MM/DD/YYYY` (например, `12/25/2023`)
  - С двузначным годом: `DD.MM.YY` или `MM/DD/YY` (например, `25.12.23`)

**Примеры**:
```datacode
global date1 = '2023-12-25'      # ISO формат
global date2 = '25.12.2023'      # Европейский формат
global date3 = '12/25/2023'      # Американский формат
```

**Особенности**:
- Хранится как строка с автоматическим распознаванием формата
- При чтении из файлов автоматически определяется тип Date

### Currency (Валюта)

**Примеры**:
```datacode
global price1 = money('100', '$0,0')      # 100 -> $100,0
global price2 = money('50.99', '€0.0')     # 50.99 -> €50.9
global price3 = money('100', '0,0 USD')    # 100 -> 100,0 USD
global price4 = money('50.99', 'EUR 0.0') # 50.99 -> EUR 50.9
```

**Особенности**:
- Хранится как строка, сохраняющая оригинальный формат

**📚 Примеры:** См. [Встроенные функции](./builtin_functions.md#moneyamount-format)

### Mixed (Смешанный тип)

- **Описание**: Используется для колонок таблиц, содержащих значения разных типов
- **Примеры**:
  ```datacode
  # Колонка может содержать как числа, так и строки
  global mixed_column = [1, 'two', 3, 'four']
  ```

**Особенности**:
- Автоматически определяется для колонок таблиц с неоднородными данными
- Генерирует предупреждения при обнаружении смешанных типов

## Совместимость типов

DataCode поддерживает автоматическую совместимость между некоторыми типами:

### Совместимые типы:

1. **Integer ↔ Float**: Целые и дробные числа совместимы и могут использоваться вместе в арифметических операциях
   ```datacode
   global x = 42      # Integer
   global y = 3.14    # Float
   global result = x + y  # Работает корректно
   ```

2. **Null ↔ любой тип**: `Null` совместим с любым типом
   ```datacode
   global value = null
   global result = value + 10  # Null совместим, но результат может быть null
   ```

3. **Одинаковые типы**: Типы совместимы сами с собой

### Несовместимые типы:

- String и Number
- Bool и Number (кроме логических операций)
- String и Bool
- Array и Object

### Автоматическое приведение типов:

DataCode выполняет некоторые автоматические преобразования:

- **Числа → Bool**: `0` → `false`, любое ненулевое → `true`
- **Строки → Bool**: пустая строка → `false`, непустая → `true`
- **Null → Bool**: всегда `false`
- **Array → Bool**: [] → `false`, не пустой список → `true`

**📚 Примеры:** [`examples/ru/02-синтаксис/booleans.dc`](../../examples/ru/02-синтаксис/booleans.dc)

## Типизация колонок таблиц

При работе с таблицами DataCode автоматически определяет типы данных колонок:

1. **Автоматический вывод типов**: При добавлении данных в таблицу система анализирует значения и определяет основной тип колонки
2. **Статистика типов**: Для каждой колонки ведется подсчет значений каждого типа
3. **Предупреждения**: При обнаружении смешанных типов генерируются предупреждения о неоднородности данных
4. **Оптимизация**: Integer и Float считаются совместимыми числовыми типами и не вызывают предупреждений

**Пример**:
```datacode
global table = table([
    [42, 'Alice', '$100'],
    [3.14, 'Bob', '€50.99']
], ['numbers', 'names', 'prices'])

# Результат:
# Колонка 'numbers': тип Float (Integer и Float совместимы)
# Колонка 'names': тип String
# Колонка 'prices': тип Currency
```

**📚 Примеры:** [`examples/ru/09-создание модели данных/`](../../examples/ru/09-создание%20модели%20данных/)

## Заключение

DataCode поддерживает богатую систему типов данных, включающую:

- **8 базовых типов**: Integer, Float, String, Bool, Date, Currency, Null, Mixed
- **2 составных типа**: Array, Object
- **4 специальных типа для таблиц**: Table, TableColumn, TableIndexer, LazyTable
- **2 типа для работы с файлами**: Path, PathPattern

Все типы автоматически определяются при загрузке данных и поддерживают гибкие преобразования для удобной работы с данными различных форматов.
