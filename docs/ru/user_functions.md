## Базовые аннотации типов

Функции могут иметь аннотации типов для своих параметров. Синтаксис:

```datacode
fn имя_функции(параметр: тип) {
    // тело функции
}
```

### Примеры

```datacode
# Функция с целочисленными параметрами
fn int_add(a: int, b: int) -> int {
    return a + b
}

# Функция с параметрами типа float
fn float_multiply(a: float, b: float) -> float {
    return a * b
}

# Функция со строковым параметром
fn greet(name: str) -> str {
    return "Привет, " + name + "!"
}

# Использование
print(int_add(5, 3))           # 8
print(float_multiply(2.5, 4.0)) # 10.0
print(greet("ДатаКод"))        # "Привет, ДатаКод!"
```

## Аннотации возвращаемого типа

Функции могут указывать свой возвращаемый тип, используя синтаксис `->`:

```datacode
fn имя_функции(параметры) -> возвращаемый_тип {
    return значение
}
```

### Примеры

```datacode
fn get_string() -> str {
    return "Это строка"
}

fn get_number() -> int {
    return 42
}

fn get_boolean() -> bool {
    return true
}

print(get_string())   # "Это строка"
print(get_number())   # 42
print(get_boolean())  # true
```

## Union типы

Union типы позволяют параметру или возвращаемому значению принимать несколько типов. Используйте оператор `|` для разделения типов:

```datacode
fn имя_функции(параметр: тип1 | тип2 | тип3) -> возвращаемый_тип {
    // тело функции
}
```

### Примеры

```datacode
# Функция, принимающая строку или целое число
fn process_value(value: str | int) -> str {
    return "Значение: " + str(value)
}

print(process_value("привет"))  # "Значение: привет"
print(process_value(42))       # "Значение: 42"

# Функция с несколькими union типами
fn flexible_add(a: int | float, b: int | float) -> float {
    return a + b
}

print(flexible_add(5, 3))      # 8.0
print(flexible_add(2.5, 3.7))  # 6.2
print(flexible_add(5, 3.5))    # 8.5

# Union тип с null
fn process_or_default(value: null | str | int = null) -> str {
    if value == null {
        return "значение по умолчанию"
    } else {
        return "получено: " + str(value)
    }
}

print(process_or_default())        # "значение по умолчанию"
print(process_or_default("тест"))  # "получено: тест"
print(process_or_default(123))     # "получено: 123"
```

### Union возвращаемые типы

Функции также могут возвращать union типы:

```datacode
fn get_value(flag: bool) -> str | int {
    if flag {
        return "строковый результат"
    } else {
        return 100
    }
}

print(get_value(true))   # "строковый результат"
print(get_value(false))  # 100
```

## Проверка типов

DataCode выполняет **проверку типов во время выполнения** при вызове функций. Если аргумент не соответствует ожидаемому типу, возникает `TypeError`.

### Поведение проверки типов

- Проверка типов происходит во время вызова функции
- Если происходит несоответствие типов, возникает `TypeError`
- Union типы позволяют использовать любой из указанных типов
- Проверка типов опциональна - функции без аннотаций типов работают как раньше

### Примеры

```datacode
fn int_add(a: int, b: int) -> int {
    return a + b
}

# Это работает
print(int_add(5, 3))  # 8

# Это вызывает TypeError
try {
    int_add(5, 3.5)  # TypeError: Argument 'b' expected type 'int', got 'float'
} catch TypeError {
    print("Ошибка типа перехвачена!")
}
```

### Обработка ошибок типов

Вы можете перехватывать `TypeError` с помощью блоков try-catch:

```datacode
fn greet(name: str) -> str {
    return "Привет, " + name + "!"
}

try {
    greet(123)  # TypeError
} catch TypeError {
    print("Предоставлен неверный тип")
}

# Или перехватить через RuntimeError (TypeError является подтипом)
try {
    greet(123)
} catch RuntimeError {
    print("Перехвачена ошибка выполнения")
}
```

## Частичная типизация

Вы можете смешивать типизированные и нетипизированные параметры в одной функции:

```datacode
# Первый параметр типизирован, второй нет
fn mixed_types(a: int, b) -> str {
    return str(a) + " + " + str(b) + " = " + str(a + b)
}

print(mixed_types(5, 3))    # "5 + 3 = 8"
print(mixed_types(5, 3.5))  # "5 + 3.5 = 8.5"

# Второй параметр типизирован, первый нет
fn first_untyped(a, b: str) -> str {
    return str(a) + " " + b
}

print(first_untyped(42, "привет"))  # "42 привет"
```

**Примечание:** Проверяются только типизированные параметры. Нетипизированные параметры принимают любой тип.

## Значения по умолчанию с типами

Вы можете комбинировать аннотации типов со значениями параметров по умолчанию:

```datacode
fn process(value: null | str | int = null) -> str {
    if value == null {
        return "значение по умолчанию"
    } else {
        return "получено: " + str(value)
    }
}

# Использование значения по умолчанию
print(process())           # "значение по умолчанию"

# Переопределение значения по умолчанию
print(process("тест"))     # "получено: тест"
print(process(123))        # "получено: 123"

# Проверка типов все еще применяется
try {
    process(true)  # TypeError: Argument 'value' expected type 'null | str | int', got 'bool'
} catch TypeError {
    print("Ошибка типа перехвачена")
}
```

## Поддерживаемые типы

Следующие типы могут использоваться в аннотациях типов:

### Базовые типы
- `int` - Целые числа
- `float` - Числа с плавающей точкой (любые числа)
- `str` или `string` - Строки
- `bool` или `boolean` - Булевы значения (true/false)
- `null` - Значение null

### Типы коллекций
- `array` - Массивы
- `tuple` - Кортежи
- `object` - Объекты/словари

### Специальные типы
- `table` - Таблицы
- `path` - Пути к файлам
- `tensor` - Тензоры ML
- `graph` - Графы вычислений
- `dataset` - Наборы данных ML
- `neural_network` - Нейронные сети
- `sequential` - Последовательные модели
- `layer` - Слои нейронных сетей

### Совместимость типов

- Значения `int` могут быть переданы в параметры `float` (int является подмножеством float)
- Значения `float` не могут быть переданы в параметры `int` (если только они не являются целыми числами)
- Union типы принимают любой из указанных типов

### Примеры

```datacode
# Тип array
fn sum_array(arr: array) -> float {
    let sum = 0.0
    for item in arr {
        sum = sum + item
    }
    return sum
}

print(sum_array([1, 2, 3, 4, 5]))  # 15.0

# Тип object
fn get_name(obj: object) -> str {
    return obj["name"]
}

let person = {"name": "Алиса", "age": 30}
print(get_name(person))  # "Алиса"

# Тип tuple
fn get_first(t: tuple) -> int {
    return t[0]
}

print(get_first((10, 20, 30)))  # 10
```

## Обработка ошибок

### TypeError

Когда происходит несоответствие типов, возникает `TypeError`. Это подтип `RuntimeError`, поэтому вы можете перехватить его любым способом:

```datacode
fn int_add(a: int, b: int) -> int {
    return a + b
}

# Перехват по конкретному типу
try {
    int_add(1, 2.5)
} catch TypeError {
    print("Ошибка типа перехвачена")
}

# Перехват по родительскому типу
try {
    int_add(1, 2.5)
} catch RuntimeError {
    print("Ошибка выполнения перехвачена")
}

# Перехват с переменной
try {
    int_add(1, 2.5)
} catch TypeError e {
    print("Ошибка:", e)
}
```

### Сообщения об ошибках

Сообщения TypeError включают:
- Имя параметра, который не прошел проверку типа
- Ожидаемый тип(ы)
- Фактический полученный тип

Пример сообщения об ошибке:
```
TypeError: Argument 'b' expected type 'int', got 'float'
```

Для union типов:
```
TypeError: Argument 'value' expected type 'str | int', got 'bool'
```

## Рекомендации

### 1. Используйте аннотации типов для ясности

Аннотации типов делают код более читаемым и самодокументируемым:

```datacode
# Ясное намерение
fn calculate_area(width: float, height: float) -> float {
    return width * height
}
```

### 2. Используйте Union типы для гибкости

Union типы обеспечивают гибкость при сохранении безопасности типов:

```datacode
# Безопасно принимает несколько типов
fn format_output(value: null | str | int) -> str {
    if value == null {
        return "null"
    } else {
        return str(value)
    }
}
```

### 3. Комбинируйте со значениями по умолчанию

Значения по умолчанию хорошо работают с union типами, включающими `null`:

```datacode
fn process(value: null | str | int = null) -> str {
    if value == null {
        return "по умолчанию"
    } else {
        return str(value)
    }
}
```

### 4. Обрабатывайте ошибки типов корректно

Используйте блоки try-catch для обработки ошибок типов:

```datacode
fn safe_process(value: str | int) -> str {
    return str(value)
}

try {
    print(safe_process("привет"))
    print(safe_process(42))
    print(safe_process(true))  # TypeError
} catch TypeError {
    print("Предоставлен неверный тип")
}
```

## Типизированные рекурсивные функции

Аннотации типов работают с рекурсивными функциями:

```datacode
# Типизированный факториал
fn factorial(n: int) -> int {
    if n <= 1 {
        return 1
    } else {
        return n * factorial(n - 1)
    }
}

print(factorial(5))  # 120

# Типизированная функция Фибоначчи
fn fibonacci(n: int) -> int {
    if n <= 1 {
        return n
    } else {
        return fibonacci(n - 1) + fibonacci(n - 2)
    }
}

print(fibonacci(7))  # 13
```

## Сравнение с нетипизированными функциями

Функции без аннотаций типов работают точно так же, как раньше - они принимают любые типы:

```datacode
# Нетипизированная функция (работает как раньше)
fn add(a, b) {
    return a + b
}

print(add(1, 2))      # 3
print(add(1, 2.5))    # 3.5
print(add("a", "b"))  # "ab"
```

Аннотации типов **опциональны** - вы можете использовать их, когда хотите безопасность типов, или опускать для гибкости.

---

## Резюме

- **Аннотации типов** обеспечивают проверку типов во время выполнения для параметров и возвращаемых значений функций
- **Union типы** (`str | int`) позволяют использовать несколько типов для гибкости
- **Проверка типов** происходит во время выполнения и вызывает `TypeError` при несоответствии
- **Частичная типизация** позволяет смешивать типизированные и нетипизированные параметры
- **Значения по умолчанию** работают с типизированными параметрами
- **Все типы** языка поддерживаются в аннотациях
- **Аннотации типов опциональны** - нетипизированные функции работают как раньше
