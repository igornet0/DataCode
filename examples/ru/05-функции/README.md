# 📦 Функции в ДатаКод

Этот раздел содержит примеры создания и использования функций.

## 📋 Содержание

### 1. `простые_функции.dc` - Простые функции
**Описание**: Демонстрирует создание и использование базовых функций.

**Что изучается**:
- Объявление функций: `fn имя(параметры) { ... }`
- Возврат значений: `return`
- Вызов функций
- Функции с параметрами и без параметров

**Запуск**:
```bash
datacode examples/ru/05-функции/простые_функции.dc
```

### 2. `рекурсия.dc` - Рекурсия
**Описание**: Демонстрирует рекурсивные функции.

**Что изучается**:
- Рекурсивные вызовы функций
- Базовые случаи рекурсии
- Примеры: факториал, числа Фибоначчи, сумма чисел

**Запуск**:
```bash
datacode examples/ru/05-функции/рекурсия.dc
```

### 3. `вложенные_функции.dc` - Вложенные функции
**Описание**: Демонстрирует использование функций в качестве аргументов других функций.

**Что изучается**:
- Вызов функций в выражениях
- Функции как аргументы других функций
- Композиция функций

**Запуск**:
```bash
datacode examples/ru/05-функции/вложенные_функции.dc
```

### 4. `типизированные_функции.dc` - Типизированные функции
**Описание**: Демонстрирует аннотации типов функций и проверку типов во время выполнения.

**Что изучается**:
- Аннотации типов для параметров: `fn add(a: int, b: int)`
- Аннотации возвращаемого типа: `-> int`
- Union типы: `str | int`, `null | str | int`
- Проверка типов и обработка TypeError
- Частичная типизация (некоторые параметры типизированы, другие нет)
- Типизированные рекурсивные функции

**Запуск**:
```bash
datacode examples/ru/05-функции/типизированные_функции.dc
```

## 🎯 Изучаемые концепции

### Объявление функций
```dc
fn имя_функции(параметр1, параметр2) {
    // код функции
    return значение
}
```

### Возврат значений
- `return значение` - возвращает значение из функции
- Функция может возвращать любое значение (число, строку, булево значение)

### Рекурсия
Рекурсивная функция вызывает сама себя. Важно определить базовый случай, чтобы избежать бесконечной рекурсии.

### Область видимости
- Параметры функции доступны только внутри функции
- Локальные переменные в функции не видны снаружи
- Глобальные переменные доступны из функций

### Аннотации типов
Функции могут иметь аннотации типов для параметров и возвращаемых значений:
```dc
fn add(a: int, b: int) -> int {
    return a + b
}
```

### Union типы
Параметры могут принимать несколько типов, используя синтаксис union:
```dc
fn process(value: str | int) -> str {
    return str(value)
}
```

### Проверка типов
- Проверка типов во время выполнения гарантирует соответствие аргументов ожидаемым типам
- TypeError возникает, когда типы не совпадают
- Union типы обеспечивают гибкость при сохранении безопасности типов
