# 📊 Создание моделей базы данных

Этот раздел демонстрирует, как построить полную модель базы данных из CSV файлов с помощью DataCode. Вы научитесь загружать данные из нескольких файлов, объединять таблицы, создавать связи и экспортировать всё в базу данных SQLite.

## 📁 Файлы в этом разделе

### Основной скрипт
- **[`загрузка_данных_модели.dc`](загрузка_данных_модели.dc)** - Полный пример, который загружает все данные из директории `model_data/`, объединяет ежемесячные и квартальные таблицы, создаёт связи и экспортирует в SQLite

   - Проверка существования файлов
   - Построение путей к файлам
   - Итерация по структуре директорий

   - Базовое объединение таблиц с `merge_tables()`
   - Объединение таблиц, загруженных из файлов
   - Обработка краевых случаев (пустые массивы, одиночные таблицы)

3. **[`03-создание_связей.dc`](03-создание_связей.dc)** - Создание связей между таблицами
   - Простые связи между двумя таблицами
   - Множественные связи
   - Связи после объединения таблиц
   - Безопасное создание связей с помощью вспомогательных функций

4. **[`04-загрузка_квартальных_данных.dc`](04-загрузка_квартальных_данных.dc)** - Загрузка квартальных агрегированных данных
   - Навигация по вложенным структурам директорий
   - Загрузка квартальных сводных файлов
   - Объединение квартальных таблиц данных

5. **[`05-объединения_таблиц.dc`](05-объединения_таблиц.dc)** - Операции объединения таблиц (JOIN)
   - INNER JOIN - совпадающие строки из обеих таблиц
   - LEFT JOIN - все строки слева с совпадающими справа
   - RIGHT JOIN - все строки справа с совпадающими слева
   - FULL JOIN - все строки из обеих таблиц
   - CROSS JOIN - декартово произведение
   - SEMI JOIN - строки слева с совпадениями (без колонок справа)
   - ANTI JOIN - строки слева без совпадений
   - JOIN с несколькими ключами
   - Практические примеры использования

Это выполнит:
1. Загрузку справочных данных (каталог продуктов, регионы, сотрудники)
2. Загрузку ежемесячных данных (продажи, инвентарь, возвраты, маркетинг)

5. Объединение всех квартальных таблиц в консолидированные таблицы
6. Создание связей между таблицами
7. Экспорт всего в базу данных SQLite `load_model_data.db`

## 📚 Изучаемые концепции

### Операции с файлами
- **`list_files(path)`** - Получить список файлов и директорий по пути
- **`read_file(path)`** - Прочитать CSV файл в таблицу
- **Конкатенация путей** - Использование оператора `/` для построения путей
- **`getcwd()`** - Получить текущую рабочую директорию

### Операции с таблицами
- **`table_create(data, headers)`** - Создать таблицу из данных
- **`merge_tables(tables_array)`** - Объединить несколько таблиц с одинаковой структурой
- **`table_headers(table)`** - Получить заголовки колонок из таблицы
- **`len(table)`** - Получить количество строк в таблице

### Связи
- **`relate(table1[col1], table2[col2])`** - Создать явную связь между двумя колонками таблиц
- Связи используются при экспорте в SQLite для создания ограничений внешних ключей
- Обе колонки должны иметь совместимые типы (String ↔ String, Number ↔ Number)
- При использовании `--build_model`, явные связи, созданные с помощью `relate()`, имеют приоритет над автоматическим определением внешних ключей
- Пример: `relate(products["product_id"], sales["product_id"])` создаёт внешний ключ от `sales.product_id` к `products.product_id`

### Операции объединения таблиц (JOIN)
- **`inner_join(left, right, left_key, right_key)`** - Возвращает совпадающие строки из обеих таблиц
- **`left_join(left, right, left_key, right_key)`** - Все строки слева + совпадающие справа
- **`right_join(left, right, left_key, right_key)`** - Все строки справа + совпадающие слева
- **`full_join(left, right, left_key, right_key)`** - Все строки из обеих таблиц
- **`cross_join(left, right)`** - Декартово произведение (все комбинации)
- **`semi_join(left, right, left_key, right_key)`** - Строки слева с совпадениями (без колонок справа)
- **`anti_join(left, right, left_key, right_key)`** - Строки слева без совпадений
- **`join(left, right, keys, type)`** - Универсальная функция объединения с параметром типа
- JOIN с несколькими ключами: `inner_join(t1, t2, [["col1", "col1"], ["col2", "col2"]])`

### Обработка ошибок
- **`try/catch/endtry`** - Обработка ошибок
- Проверка на null значения перед операциями
- Валидация структур таблиц перед объединением

## 💡 Лучшие практики

1. **Всегда проверяйте на null** - Перед использованием объединённых таблиц проверьте, не являются ли они null
2. **Используйте try-catch** - Оберните операции с файлами в блоки try-catch
3. **Валидируйте структуру** - Убедитесь, что таблицы имеют совместимые структуры перед объединением
4. **Создавайте связи после объединения** - Связи должны создаваться после объединения всех таблиц
5. **Используйте вспомогательные функции** - Создавайте вспомогательные функции для общих операций (например, безопасное создание связей)
